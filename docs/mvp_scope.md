# Границы MVP ЖР-продукта

## Продуктовая цель

MVP должен дать пользователю простой способ разобрать журнал регистрации 1С, загрузить события в ClickHouse и быстро искать их через Event Explorer.

Фокус MVP — удобный разбор и поиск, а не расследования и не управление сервером 1С.

## Входит

1. Получение событий ЖР через штатную утилиту `ibcmd eventlog export`.
2. Поддержка JSON-потока из stdout.
3. Локальное сохранение сырых событий в SQLite.
4. Нормализация основных полей ЖР.
5. Запись событий в ClickHouse.
6. Хранение raw-представления события.
7. HTTP-статус сервиса.
8. Базовая схема для будущего Event Explorer.

## Не входит

1. Кластерный сбор с нескольких серверов 1С.
2. Центральный приёмник событий.
3. PostgreSQL-очередь распределённого разборщика.
4. Управление несколькими агентами.
5. Корреляция ЖР + ТЖ.
6. Автоматическое определение причин TLOCK/SDBL/EXCP/TTIMEOUT/TDEADLOCK.
7. Investigation Result / Episode / Pattern.
8. Рекомендации и ИИ-разбор.
9. Контроль размера каталога ЖР, очистка или сокращение ЖР.

## Коммерческая версия

В коммерческую версию выносятся кластерный сбор, централизованное управление, связь ЖР с ТЖ, расследования, рекомендации и ИИ-модуль.
